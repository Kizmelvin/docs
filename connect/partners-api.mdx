The **Partners API** lets you integrate with Mono Connect **without using the Mono Connect widget**. You get a fully custom, end-to-end experience: your UI, your branding, and programmatic control over account linking. Ideal for white-label products, backend-triggered flows, or regulated environments.

## Benefits

- **Full control over UX** – Custom UI and flow; no Mono modals or widget.
- **Backend-only flows** – Initiate and manage linking via API (e.g. trigger via SMS, email, or server).
- **Multi-tenant and white-label** – Different brands or clients can have their own linking experience.
- **Strict environments** – Fit environments that require strict control over branding and data flow.
- **Channels beyond browser/mobile** – Support WhatsApp, USSD, kiosks, or other channels.
- **Consent and tokens** – Implement granular consent and custom token management.

## When to use it

| Use case                    | Example                                                                                             |
| --------------------------- | --------------------------------------------------------------------------------------------------- |
| **Custom or native UX**     | A fintech wants users to link their bank from within their own app screens, with their own styling. |
| **White-labeled platforms** | A SaaS for loan companies gives each client a branded portal where end-users link accounts.         |
| **Backend-triggered flows** | A credit engine sends users a link via SMS to start linking; the flow is managed via API.           |

## How it works

1. **Create a session** – Use the Partners API to create a session (e.g. [create-session](/api-reference/bank-data)) with the desired scope and customer reference.
2. **Send the user to the bank** – The user completes authentication with their bank (via the link or flow returned by the API).
3. **Receive webhooks** – Mono sends the same Connect webhooks (e.g. account_connected) so you can store the account ID and update your UI.
4. **Reauth when needed** – Use the [Partners API reauth](/api-reference/bank-data) (or reauth login) when the user must re-authorise access.

Sandbox and live use the same flow; use sandbox keys for testing. For detailed request/response parameters, see the [Partners API reference](/api-reference/bank-data) and related endpoints (login, reauth-session, commit-session, etc.).

## Pricing and access

- **Pricing** – Same as Connect (per connected account). See the [Mono pricing page](https://mono.co/pricing).
- **Access** – The Partners API is available to partners with the Mono Partner API add-on. Contact Mono or your account manager to enable it.

## Sandbox and webhooks

- **Sandbox** – All Partners API endpoints work in sandbox so you can test without live bank connections.
- **Webhooks** – Mono sends the same webhook events as for widget-based Connect (e.g. account_connected, account_updated, account_disconnected). Configure your webhook URL in the dashboard.

For step-by-step integration and reauth, see [Connect integration](/essentials/mono/connect-integration) and [Partners API](/product-abilities/partners-api). For a high-level alternative (Connect Link with generated link), see [Connect integration](/essentials/mono/connect-integration).
