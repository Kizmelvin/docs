Mono Connect lets your users securely link their user bank accounts so they can retrieve accurate financial data. This page summarises what Connect offers and where to go for implementation details.

This guide will show you how to integrate the Mono Connect API into your product, with clear steps and sample requests for the entire process.

<Tip>
  **Prerequisites**

  - [Sign up](https://app.mono.co/signup) on the Mono Dashboard and complete KYC.
  - [Create a Connect app](/quickstart/create-an-app) with the right **product scopes** and copy your **secret key** (for `mono-sec-key` on the server). See [How to create an app](https://support.mono.co/en/articles/7054553-how-to-create-an-app) if you need help choosing scopes.
  - Add a **webhook URL** in the dashboard (try [webhook.site](https://webhook.site/) for a quick test URL) and follow [Webhook events](/utilities/webhooks) to verify payloads.
</Tip>

## Step-by-step integration guide

<Steps>
  <Step title="Call the initiate linking endpoint">
    To set up recurring payments or subscriptions, the first step is to create a
    Customer on Mono. This endpoint streamlines onboarding, ensures you are
    KYC-compliant, and provides a crucial identifier for seamless integration
    with Mono's endpoints.

    <b> Sample request</b>
    <CodeGroup>
    ```bash Curl
    curl --request POST \
     --url https://api.withmono.com/v2/accounts/initiate \
     --header 'accept: application/json' \
     --header 'content-type: application/json' \
     --header 'mono-sec-key: string' \
     --data '{
    "customer": {
        "name": "Samuel Olamide",
        "email": "samuel@neem.com"
    },
    "meta": { "ref": "99008877TEST"},
    "scope": "auth",
    "redirect_url": "https://mono.co"

}

````

```javascript JavaScript
const response = await fetch('https://api.withmono.com/v2/accounts/initiate', {
method: 'POST',
headers: {
'accept': 'application/json',
'content-type': 'application/json',
'mono-sec-key': process.env.MONO_SEC_KEY,
},
body: JSON.stringify({
customer: { name: 'Samuel Olamide', email: 'samuel@neem.com' },
meta: { ref: '99008877TEST' },
scope: 'auth',
redirect_url: 'https://mono.co',
}),
});
const data = await response.json();

````

  <Step title="Finding a reliable way to identify customers who can pay back the loan">
  Using the Mono Income endpoint: The [Mono Income endpoint](/connect/income) helps you determine a user's income pattern.

To initiate the income endpoint, send a GET request to the following endpoint:

<b>Sample request body:</b>

<CodeGroup>
```bash Curl
curl --request GET \
  --url https://api.withmono.com/v2/accounts/{accountId}/income \
  --header 'Content-Type: application/json' \
  --header 'accept: application/json' \
  --header 'mono-sec-key: string'
```
```javascript JavaScript
const accountId = 'YOUR_ACCOUNT_ID';
const response = await fetch(`https://api.withmono.com/v2/accounts/${accountId}/income`, {
  method: 'GET',
  headers: {
    'Content-Type': 'application/json',
    'accept': 'application/json',
    'mono-sec-key': process.env.MONO_SEC_KEY,

});
const data = await response.json();
```

  <Step title="Determining the safest loan amount to extend to the customer, with the highest probability of being paid back">
Using the Mono Creditworthiness endpoint: The [Mono Creditworthiness endpoint](/connect/credit-worthiness) allows you to assess the repayment capacity of a user by evaluating the debt-to-income ratio against their transactions, credit history, and the requested loan amount, interest rate, and tenor.

To call the creditworthiness endpoint, see sample request below:

<b>Sample request body:</b>

<CodeGroup>
```bash Curl
curl --request POST \
  --url https://api.withmono.com/v2/accounts/{accountId}/creditworthiness \
  --header 'accept: application/json' \
  --header 'content-type: application/json' \
  --header 'mono-sec-key: string' \
  --data '{
  "bvn": "12345678901",
  "principal": 30000000,
  "interest_rate": 5,
  "term": 12,
  "run_credit_check": true
}'
```

## Common issues & debugging tips

Webhook events not being received:

Webhooks events sometimes may not be received because of network issues with the bank which might prevent account linking from happening successfully. When this happens, you can call the endpoint that returns all linked accounts to confirm if the account was linked successfully or not.

Sample request:

<CodeGroup>
```bash Curl
curl --request GET \
  --url https://api.withmono.com/v2/accounts \
  --header 'accept: application/json' \
  --header 'mono-sec-key: string'
```
```javascript JavaScript
const response = await fetch('https://api.withmono.com/v2/accounts', {
  method: 'GET',
  headers: {
    'accept': 'application/json',
    'mono-sec-key': process.env.MONO_SEC_KEY,
  },
});
const data = await response.json();
```
</CodeGroup>

This is a sample of the response you will receive so you can verify all customers that have actually linked their accounts successfully.

<b>Sample response:</b>
<CodeGroup>
```bash JSON
{
    "status": "successful",
    "message": "Data retrieved successfully",
````

Unavailable financial data when calling an endpoint

This may happen if account linking failed or if no data is available.

To debug: After linking an account, the <code>ACCOUNT-UPDATED</code> webhook event received contains a <code>META</code> object with a data status of: available, processing, or failed.

Calling financial data endpoints while status is processing or failed won’t return any data. data_status must be available to obtain useful results.
