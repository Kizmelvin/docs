As a credit business, making accurate risk assessment decisions is often challenging. This is because finding reliable ways to retrieve customers’ financial data, determine the maximum amount to lend to them, or assess their capacity to repay loans is hard.

The Mono Connect API product helps you solve these problems by making it easier to access customers’ verified bank statements and better understand their financial behaviour. This empowers you with accurate insights into users’ cash flow and finance patterns, so you can make data-informed lending decisions and lend the right amount of money to the right person.

This quick start guide will show you how to integrate the Mono Connect API into your product, with clear steps and sample requests for the entire process.

## Prerequisites

To get started, you'll need your Mono secret key and a webhook URL. To get these, follow the steps below:

[Sign up](https://app.mono.co/signup) on the Mono Dashboard and complete KYC.
[Create a Connect App](/quickstart/create-an-app) to generate your Mono Secret Key (this will be used for backend integrations or postman tests). Note: When creating your Connect app, you are expected to choose the product scopes for the app. You can check [this article](https://support.mono.co/en/articles/7054553-how-to-create-an-app) to see how to do that correctly.
Configure a webhook URL and add it to your dashboard. You can use [this site](https://webhook.site/) to quickly generate a webhook URL, and follow [this guide](/utilities/webhooks) to integrate it properly.

```bash Curl
  --header 'mono-sec-key: string'
```

```javascript JavaScript
const accountId = 'YOUR_ACCOUNT_ID';
```

Using the Mono Creditworthiness endpoint: The [Mono Creditworthiness endpoint](/product-abilities/credit-worthiness) allows you to assess the repayment capacity of a user by evaluating the debt-to-income ratio against their transactions, credit history, and the requested loan amount, interest rate, and tenor.

```bash Curl
  "interest_rate": 5,
```

## Common issues & debugging tips

Webhook events not being received:

Webhooks events sometimes may not be received because of network issues with the bank which might prevent account linking from happening successfully. When this happens, you can call the endpoint that returns all linked accounts to confirm if the account was linked successfully or not.

Sample request:

<CodeGroup>
```bash Curl
curl --request GET \
  --url https://api.withmono.com/v2/accounts \
  --header 'accept: application/json' \
  --header 'mono-sec-key: string'
```
```javascript JavaScript
const response = await fetch('https://api.withmono.com/v2/accounts', {
  method: 'GET',
  headers: {
    'accept': 'application/json',
    'mono-sec-key': process.env.MONO_SEC_KEY,
  },
});
const data = await response.json();
```
</CodeGroup>

This is a sample of the response you will receive so you can verify all customers that have actually linked their accounts successfully.

<b>Sample response:</b>
<CodeGroup>

Unavailable financial data when calling an endpoint

This may happen if account linking failed or if no data is available.

To debug: After linking an account, the <code>ACCOUNT-UPDATED</code> webhook event received contains a <code>META</code> object with a data status of: available, processing, or failed.

Calling financial data endpoints while status is processing or failed won’t return any data. data_status must be available to obtain useful results.
