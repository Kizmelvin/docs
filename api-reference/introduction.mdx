All Mono API requests use the same base URL and authentication. Use the **`mono-sec-key`** header with your secret key from the [Mono dashboard](https://app.mono.co/apps).

## Base URL

```
https://api.withmono.com
```

## Authentication

Send your secret key in the **`mono-sec-key`** header on every request. Do not use it in client-side or public code.

## Quick example: Initiate account linking

**Request**

<CodeGroup>
  ```bash Curl
  curl -X POST 'https://api.withmono.com/v2/accounts/initiate' \
    -H 'Content-Type: application/json' \
    -H 'mono-sec-key: YOUR_SECRET_KEY' \
    -d '{"customer":{"name":"Customer Name","email":"customer@example.com"},"scope":"auth","redirect_url":"https://yourapp.com/callback"}'
  ```
  ```javascript Node
  const response = await fetch('https://api.withmono.com/v2/accounts/initiate', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'mono-sec-key': process.env.MONO_SEC_KEY
    },
    body: JSON.stringify({
      customer: { name: 'Customer Name', email: 'customer@example.com' },
      scope: 'auth',
      redirect_url: 'https://yourapp.com/callback'
    })
  });
  const data = await response.json();
  ```
</CodeGroup>

**Response (200)**

```json
{
  "id": "account_xxx",
  "code": "auth_code_xxx",
  "link": "https://connect.withmono.com/..."
}
```
