Bank Data (Mono Connect) endpoints let you link accounts, sync transactions, fetch balance and identity, and use enrichment. Base URL: **`https://api.withmono.com`**. Authenticate with **`mono-sec-key`**.

---

## Initiate account linking

**POST** `api.withmono.com/v2/accounts/initiate`

Generates a link for users to connect their bank account.

**Request**

<CodeGroup>
  ```bash Curl
  curl -X POST 'https://api.withmono.com/v2/accounts/initiate' \
    -H 'Content-Type: application/json' \
    -H 'mono-sec-key: YOUR_SECRET_KEY' \
    -d '{"customer":{"name":"Customer","email":"customer@example.com"},"scope":"auth","redirect_url":"https://yourapp.com/callback"}'
  ```
  ```javascript Node
  const response = await fetch('https://api.withmono.com/v2/accounts/initiate', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'mono-sec-key': process.env.MONO_SEC_KEY
    },
    body: JSON.stringify({
      customer: { name: 'Customer', email: 'customer@example.com' },
      scope: 'auth',
      redirect_url: 'https://yourapp.com/callback'
    })
  });
  const data = await response.json();
  ```
</CodeGroup>

**Response (200)**

```json
{
  "id": "account_xxx",
  "code": "auth_code_xxx",
  "link": "https://connect.withmono.com/..."
}
```

## Account details

**GET** `api.withmono.com/v1/accounts/{id}`

Returns account metadata, balance, and data status for a linked account.

**Path params:** `id` (string, required) – Account ID from the linking flow.

**Request**

<CodeGroup>
  ```bash Curl
  curl -X GET 'https://api.withmono.com/v1/accounts/account_id_here' \
    -H 'mono-sec-key: YOUR_SECRET_KEY'
  ```
  ```javascript Node
  const accountId = 'account_xxxxxxxx';
  const response = await fetch(`https://api.withmono.com/v1/accounts/${accountId}`, {
    method: 'GET',
    headers: { 'mono-sec-key': process.env.MONO_SEC_KEY }
  });
  const data = await response.json();
  ```
</CodeGroup>

**Response (200)**

```json
{
  "account": {
    "id": "account_xxxxxxxx",
    "name": "Account Name",
    "balance": 15000,
    "currency": "NGN",
    "type": "deposit",
    "data_status": "available"
  }
}
```

## Transactions

**GET** `api.withmono.com/v1/accounts/{id}/transactions`

Lists transactions for a linked account (paginated).

**Path params:** `id` (string, required) – Account ID.

**Request**

<CodeGroup>
  ```bash Curl
  curl -X GET 'https://api.withmono.com/v1/accounts/account_id_here/transactions?page=1&limit=20' \
    -H 'mono-sec-key: YOUR_SECRET_KEY'
  ```
  ```javascript Node
  const accountId = 'account_xxxxxxxx';
  const response = await fetch(`https://api.withmono.com/v1/accounts/${accountId}/transactions?page=1&limit=20`, {
    method: 'GET',
    headers: { 'mono-sec-key': process.env.MONO_SEC_KEY }
  });
  const data = await response.json();
  ```
</CodeGroup>

**Response (200)**

```json
{
  "data": [],
  "meta": { "page": 1, "total": 0 }
}
```

## Unlink account

**POST** `api.withmono.com/v1/accounts/{id}/unlink`

Revokes the link for a connected account.

**Path params:** `id` (string, required) – Account ID.

**Request**

<CodeGroup>
  ```bash Curl
  curl -X POST 'https://api.withmono.com/v1/accounts/account_id_here/unlink' \
    -H 'mono-sec-key: YOUR_SECRET_KEY'
  ```
  ```javascript Node
  const accountId = 'account_xxxxxxxx';
  const response = await fetch(`https://api.withmono.com/v1/accounts/${accountId}/unlink`, {
    method: 'POST',
    headers: { 'mono-sec-key': process.env.MONO_SEC_KEY }
  });
  const data = await response.json();
  ```
</CodeGroup>

**Response (200)**

```json
{
  "status": "success",
  "message": "Account unlinked successfully"
}
```
